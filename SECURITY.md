# 安全策略

## 报告漏洞

如果你发现安全漏洞，**请不要创建公开 Issue**。

请通过 GitHub 的 [非公开漏洞报告](https://github.com/JananZZZ/1shot-cc/security/advisories/new) 功能提交，或直接联系项目维护者。

我们会在 48 小时内确认收到报告，并在 7 天内提供初步评估。

## 安全原则

1shot-CC 的设计遵循以下安全原则：

- 🔒 **本地运行** — 所有操作在你的电脑上完成，不上传任何数据
- 🔑 **密钥安全** — API Key 仅存储在你的本地 `~/.claude.json` 中，从未硬编码或上传
- 🛡️ **管理员权限** — 仅在安装系统级软件（Node.js、Git）时才需要，且会明确告知
- 🌐 **网络下载** — 安装包来源均为官方或可信镜像，下载前会校验完整性

## 依赖项

我们会定期审查依赖项的安全性。如果你发现某依赖项存在已知漏洞，请告知我们。
